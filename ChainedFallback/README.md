# ChainedFallback

게임 설정 시스템에서 여러 설정 소스의 우선순위에 따라 최종 값을 결정하는 프로그램을 작성하세요.

설정 소스는 우선순위가 높은 순서대로 사용자 설정, 프로필 설정, 전역 설정, 하드코딩 기본값 4단계입니다. 
`??` 체이닝으로 첫 번째 non-null 값을 선택하고, `??=`로 결정된 값을 캐시에 저장하세요.

## 요구사항

1. 4개의 설정 항목에 대해 각각 4단계 소스를 선언하세요:
   - **플레이어 이름**: 사용자 설정 = null, 프로필 설정 = "프로필유저", 전역 설정 = null, 기본값 = "Player"
   - **서버 주소**: 사용자 설정 = null, 프로필 설정 = null, 전역 설정 = null, 기본값 = "localhost"
   - **언어**: 사용자 설정 = "English", 프로필 설정 = "Korean", 전역 설정 = "Japanese", 기본값 = "Korean"
   - **볼륨** (int?): 사용자 설정 = null, 프로필 설정 = null, 전역 설정 = 70, 기본값 = 50

2. 각 항목에 `??` 체이닝을 적용하여 최종 값을 결정하세요:
   - `userSetting ?? profileSetting ?? globalSetting ?? defaultValue`

3. 결정된 최종 값을 `??=`로 캐시 변수에 저장하세요:
   - 캐시 변수는 null로 초기화
   - `??=`로 한 번만 저장되는 것을 확인 (두 번째 `??=`는 무시됨)

4. 캐시에서 값을 읽어 최종 결과를 출력하세요.

## 예상 실행 결과

```
=== 설정 결정 ===
플레이어 이름: 프로필유저 (프로필 설정에서 가져옴)
서버 주소: localhost (기본값에서 가져옴)
언어: English (사용자 설정에서 가져옴)
볼륨: 70 (전역 설정에서 가져옴)

=== 캐시 저장 (??=) ===
캐시 저장 전: null
첫 번째 ??= 적용 후: 프로필유저
두 번째 ??= 적용 후: 프로필유저

=== 최종 설정 ===
플레이어 이름: 프로필유저
서버 주소: localhost
언어: English
볼륨: 70
```
